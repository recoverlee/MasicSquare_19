## 추천

### 1) 혼자 쓰고, 변경이 작을 때  
**`main`에서 바로 커밋**해도 충분합니다.  
커밋 메시지는 예: `docs: README …` 처럼 `docs:` 접두만 맞춰 두면 나중에 로그 보기 좋습니다.

### 2) PR을 쓰거나, README를 크게 바꿀 때  
짧게 살아 있는 전용 브랜치가 좋습니다.

- 이름 예: `docs/readme-update`, `docs/readme-structure`
- 흐름: 브랜치 생성 → README 수정 → 커밋 → PR → `main` 머지

이 프로젝트는 문서·학습용 성격이 강하므로 **`docs/` 접두 + 한 줄로 무슨 작업인지**만 적어도 협업·히스토리 모두 읽기 쉽습니다.

### 3) 피하면 좋은 것  
- README만 바꾸는데 `feature/`처럼 기능 브랜치 이름을 쓰는 것 (의미가 안 맞음)  
- 한 브랜치에 README와 무관한 코드 변경을 섞는 것
